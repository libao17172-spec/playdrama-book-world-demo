# Phaser 可交付书境 Demo 实施计划

## 目标

把《纳瓦尔宝典》演示场景重建为一个真实可玩的 2.5D 世界。角色移动、镜头、知识节点和场景物件共享同一套世界坐标；玩家必须走近目标才能互动；完成一条两分钟学习路线后得到明确的章节完成反馈。现有 React 知识详情和知识图谱保留，内容继续由内容包配置驱动。

## 验收门槛

- W、A、S、D 和方向键移动时，角色朝向与移动方向一致；松键后停止。
- 鼠标或触屏点击道路可寻路移动，移动端有可用的方向控制与互动按钮。
- 镜头跟随角色，角色移动会看到地图中的新区域；知识节点使用世界坐标，不随屏幕固定。
- 只有进入节点有效距离后才能打开知识内容；远处点击只提供路线引导。
- 道路范围连续可通，碰撞物可见；提供返回安全位置能力。
- 至少三处核心知识点包含解释、现实例子、行动建议和思考题。
- 完成指定学习路线后出现章节总结，形成进入、行走、互动、学习、完成的演示闭环。
- Phaser 使用 AUTO 渲染，WebGL 不可用时能使用 Canvas；React 页面和弹窗不依赖隐藏的 Three.js 位移状态。
- 桌面端与手机端均完成浏览器实测，单元测试、端到端测试和生产构建全部通过。

## 实施任务

### 1. 世界规则测试

先为角色位置限制、节点距离、互动判定、路线进度和完成条件写失败测试，再实现纯函数世界模型。

### 2. Phaser 场景桥接

安装 Phaser，新增 React 容器和场景工厂。场景从内容包读取地图、出生点、节点和路线配置，通过事件把角色位置、附近节点、发现状态和互动请求传回 React。

### 3. 真实空间玩法

把背景、角色、节点、障碍物和相机放入同一世界坐标。实现键盘移动、点击移动、方向动画、镜头跟随、近距离提示、路线引导和安全复位。

### 4. 学习闭环

用《纳瓦尔宝典》的阅读、专长、责任三个知识点组成首条路线。走近并完成学习后更新进度，最终展示本章知识总结和可演示的完成状态。

### 5. 演示界面收口

移除未接入能力的显眼提示，压缩遮挡画面的面板，增加任务目标、触屏控制和互动反馈。保留知识图谱，确保视觉层级不妨碍行走。

### 6. 交付验证

运行单元测试、端到端测试和生产构建；用真实浏览器分别验证桌面和手机尺寸，保存截图并逐条对照验收门槛。通过后合并并部署。
